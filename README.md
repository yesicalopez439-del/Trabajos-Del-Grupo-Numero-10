DESCRIPCIÓN DE TRABAJOS:

TAREA 2: El taller  es un trabajo en equipo que esta enfocado en el Diseño de una Base de Datos en el Modelo Entidad-Relación (E-R) para el Caso de Estudio: Zoológico.
Su objetivo es aplicar el Modelo Entidad-Relación (E-R) para modelar la información del zoológico (especies, empleados, itinerarios, etc.).
El producto clave es una propuesta de Modelo Conceptual básico que identifica entidades, atributos, y relaciones con sus respectivas cardinalidades.
Dicho modelo conceptual debe ser representado obligatoriamente mediante un Diagrama de Entidad-Relación de Chen.
Adicionalmente, se requiere una investigación sobre diferentes modelos E-R y la presentación de un informe final junto con un video de sustentación en un repositorio Git.

TAREA 3: El taller TIA-03 es una actividad en equipo enfocada en la Normalización y Diccionario de Datos dentro del proyecto más amplio de "Arquitectura de una base de datos
para el servicio de Hospitalización del Sistema de Salud del Departamento de Antioquia, Colombia".Esta tarea se centra en la elaboración del Modelo Lógico y tiene como actividades
principales: Normalización de Datos: Realizar el proceso completo de normalización de las tablas de la base de datos, incluyendo la Primera Forma Normal (1FN), Segunda Forma Normal (2FN)
y Tercera Forma Normal (3FN).Diccionario de Datos: Creación y elaboración del Diccionario de Datos como parte de la documentación.Inventarios y Diagrama E-R: Elaborar el inventario de 
Entidades y Relaciones, además de construir el Diagrama Entidad-Relación correctamente según los requerimientos de hospitalización.

TAREA 5: El taller TIA-05 lleva por nombre DDL - Diccionario de Datos Físico y se enfoca en la fase de Implementación de la Base de Datos (Modelo Físico) dentro del proyecto general.
La TIA-05 se centra en transformar el diseño lógico previo en una estructura física dentro de un Sistema Gestor de Base de Datos (SGBD). Las actividades principales incluyen:
Preparación: Corregir el Diagrama Entidad-Relación y revisar el proceso de normalización previo.Diccionario Físico: Elaborar el Diccionario de Datos Físico, asegurando que las tablas 
resultantes tengan los mismos nombres que las entidades y relaciones del proceso de normalización.Scripts DDL de Creación: Generar los Scripts DDL (Data Definition Language) para la creación de todas
las tablas (CREATE TABLE).Scripts DDL de Modificación: Crear Scripts DDL para la modificación de las tablas (ALTER TABLE), los cuales deben ejecutarse después de la creación.
